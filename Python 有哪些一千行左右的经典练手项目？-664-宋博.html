感谢评论区的知友提醒，构造HTTP包可以用requests库，会轻便很多。这个库的宣传语也很有趣
Requests: HTTP for Humans
www. python - requests .org/
不过requests库并不是Python的自带库，使用前需要进行安装。Windows或Linux用户可以用pip进行安装。（前提是电脑已装过pip包管理工具）
在命令行下输入： 
pip install requests
即可完成安装
有了requests后，构造POST请求就方便得多。 
import requests
username = your_username
password = your_password
payload ={action:login,username:username,password:password,ac_id:3,type:1,wbaredirect:http://net.zju.edu.cn,
mac:undefined,user_ip:,is_ldap:1,local_auth:1}
url = "https://net.zju.edu.cn/cgi-bin/srun_portal"
res = requests.post(url, data = payload)
#print response
print res.text
构造其他类型的HTTP请求也很方便 
import requests
#GET
res = requests.get("http://www.baidu.com")
#POST
res = requests.post("http://www.baidu.com")
#DELETE
res = requests.delete("http://www.baidu.com")
#PUT
res = requests.put("http://www.baidu.com")

换成电脑答题，关于用Python写学校无线登陆脚本的一些步骤。

〇、查看登陆页面的结构
以我们学校为例：
在连接学校wifi但没有登陆时，无论你在地址栏敲入什么网址，都会被重定向到 http:// net.zju.edu.cn 这个页面。在此页面输入用户名密码并登陆后，才能上网。

一、用Chrome找到完成登陆操作的数据包
在Chrome按下F12审查元素，选择netwrok选项，勾选保存日志(preserve log)，因为有可能登陆成功后页面发生跳转，这时需要我们保存日志。输入用户名密码后，单击登陆。这时你会看到浏览器向服务器发送了一堆数据包：

（从上图的密码框的长度来看，我们能得出一个规律： 程序员的密码一般都很长 ）
其中我们需要找到包含用户名密码的数据包，这个包最为关键，因为它完成了登陆操作。上图中红框内的就是这个包。
一个技巧，一般包含用户名密码的数据包都会以POST方法发送。因为它是浏览器递交数据至服务器，而且是敏感信息，不应出现在url中。
从REST HTTP的层面理解，它更改了服务器的状态。

二、分析数据包的格式
我们点开这个包，会看到form data中出现了username，password字段，这就是我们想要的数据包。

在这个包的header中我们也可以找到这个包的目的地址，截图中没有显示出来，它是：
https:// net.zju.edu.cn/cgi-bin/ srun_portal

三、用Python构建符合条件的数据包，发送到服务器对应地址
直接上代码了，重点是构造和第二步抓包得到的一样的POST数据。 
#Import urlencode() in this package to encode post data
import urllib
#Import http relevant functions 
from urllib2 import Request, urlopen, URLError, HTTPError
username = your_username
password = your_password
data ={action:login,username:username,password:password,ac_id:3,type:1,wbaredirect:http://net.zju.edu.cn,
mac:undefined,user_ip:,is_ldap:1,local_auth:1}
data = urllib.urlencode(data)
try:
    req = Request("https://net.zju.edu.cn/cgi-bin/srun_portal")
    response = urlopen(req,data, timeout = 10)	
    content = response.read()
    response.close()
except URLError, e:
    if hasattr(e, reason):
        info = [ERROR] Failed to reach the server.\nReason:  + str(e.reason)
    elif hasattr(e, code):
        info = [ERROR] The server couldn\t fullfill the request.\nError code:  +str(e.code)
    else:
        info = [ERROR] Unknown URLError
        cPrint(info, COLOR.RED)
        return False

except Exception:
    import traceback
    print "Generic exception: " + traceback.format_exc()
    return False

----原答案
私以为千行的Python算是大项目了吧，在学校里的话。其实看似简单的项目，深挖的话，可以实现的功能有很多。

如果题主是学生的话，可以写写学校无线的自动登录脚本，其实核码没几行，就是构造一个GET或POST请求，发到验证的地址就好。不仅提高上网效率，还能分享给校内的同学们。

如果深挖这个技术的话……

需要用Chrome抓包(或其他抓包程序)，构造正确的request;

处理HTTP返回码，处理异常(超时，密码错误，服务器无法访问等);

处理COOKIE【如果需要的话】;

让程序记住用户密码，以供下次自动登录的话，要用到数据库存储;

密码最好加密存储，实现个AES吧;

AES的密钥如何选取?各个用户的密钥别用一样的，那获取本机网卡的MAC地址；

索性把自动发现ZJUWLAN(我们学校的wifi名)，自动连接wifi，断线重连也搞定了；

答主实现了以上的功能，代码量才六七百行。

https:// github.com/zjusbo/ZJUWL AN_AUTO_LOGIN

破解其他同学的wifi账号，因为账号名就是学号，密码暴力枚举。你会发现好多同学的密码真的很脆弱。
